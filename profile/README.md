# Critical Software Risk Register

Critical infrastructure worldwide depends on open source components 
maintained by dangerously small teams — often one or two people. 
When a maintainer stops, the software doesn't get patched. 
In OT environments, that means risk that can't be patched away.

This project maps that risk. The whitepaper is the first public artifact. 
The registry follows.
